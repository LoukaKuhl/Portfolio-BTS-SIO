Bilan de stage

Mon stage chez Agence418 — ce que j'ai appris, les défis rencontrés et ce que j'en retiens.

Ce que j'ai appris

  • Conception et déploiement d'une API REST complète en TypeScript/Express
  • Gestion d'une base de données Supabase avec politiques RLS
  • Mise en place d'un Service Worker pour le mode hors ligne (PWA)
  • Déploiement via Docker sur une plateforme cloud (Render)
  • Travail en autonomie sur un projet réel avec contraintes métier

Difficultés rencontrées

  • Prise en main de TypeScript strict en conditions réelles
  • Gestion des conflits de cache avec le Service Worker
  • Configuration Docker multi-stage pour optimiser la taille de l'image
  • Sécurisation des routes API avec les politiques RLS de Supabase

Ce que je referais différemment

  • Rédiger les tests unitaires dès le début du développement
  • Mieux documenter l'API dès la conception (Swagger/OpenAPI)
  • Prévoir plus tôt la stratégie de cache offline

Veille Technologique

Qu'est-ce que la veille technologique ?

La veille technologique consiste à surveiller régulièrement les nouveautés, les évolutions techniques et l'émergence de nouveaux outils afin de maintenir mes compétences à jour et de proposer des architectures logicielles modernes et sécurisées.

Méthodologie & Outils de collecte

Pour structurer ma veille de manière efficace, j'utilise un écosystème d'outils professionnels :

  • Agrégateurs de flux (Feedly) : Pour centraliser les articles de blogs d'ingénieurs techniques.
  • Plateformes communautaires : Suivi des tendances sur GitHub Explore, Dev.to, et Medium (Tech blogs).
  • Newsletters spécialisées : JavaScript Weekly, Node Weekly et TLDR Web Dev.

Sujet de ma veille

Thématique principale : L'écosystème JavaScript Moderne (Back-end & Cloud) et les architectures d'applications décentralisées

Ma veille s'articule autour de deux axes stratégiques liés à mes projets de stage :

Thème 1 : Évolutions de Node.js et du Back-end TypeScript

  • TypeScript : Suivi des mises à jour du langage pour sécuriser le typage des API (Dev.to / TypeScript Blog).
  • Frameworks & APIs : Évolution des performances d'Express et alternatives émergentes (Fastify, NestJS).
  • Bases de données BaaS : L'essor de Supabase et l'intégration des politiques de sécurité de niveau ligne (RLS) avec PostgreSQL.

Thème 2 : Performance Web, PWA et Conteneurisation

  • Progressive Web Apps (PWA) : Nouvelles API des navigateurs pour la gestion avancée du cache et des Services Workers en mode déconnecté.
  • Conteneurisation (Docker) : Bonnes pratiques d'écriture de Dockerfiles légers (Multi-stage builds) pour optimiser le déploiement.
  • PaaS / Déploiement : Évolution des services de Cloud modernes comme Render et Vercel face aux hébergements traditionnels.

Apports pour ma formation

Cette veille active m'a permis d'acquérir les connaissances théoriques indispensables à la réalisation de mon application CodeGrimoire. Elle me permet d'anticiper les besoins du marché et de choisir des technologies adaptés, performantes et pérennes.
